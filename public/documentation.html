myFlix API Documentation

This is an REST API for application called myFlix that interacts with database and store data about different movies

You will be able to Sing up,update your personal information,and create a list of your favorite movies


/users

Description:This endpoint allows users to register.Endpoint returns a JSON object with few key values.For key "name" it returns string,for "favorite movies" array, and for an "id" string.

Query Parameters: None

Expected request data: JSON object with data about user. Required: "name"

Example:

{
"name": "John Smith",
"favorite movies": []
}

Expected response data: JSON object with data about user which includes automatic generated ID

Example:

{
"name": "John Smith",
"id": "8a757bb9-2eec-46bf-86f0-443ff742cf3a"
}

Response status codes:

Created- New user has been added

Bad request- User need names

Internal server error- Something went wrong

/users/:id

Description: This endpoint allows users to update their info,such as name.Endpoint returns a JSON object with few key values.For key "name" it returns string,for "favorite movies" array, and for an "id" integer.

Query Parameters: :id,

Expected request data: JSON object with data about user. Required: :id, "name"

Example:

localhost:8080/users/2


{
"name": "John Doe"
}

Expected response data: JSON object with data about user which includes,"id" with integer value, "name" with string and "favorite movies" with an array.

Example:

{
"id": 2,
"name": "John Doe",
"favoriteMovies": [
"Venom"
]
}

Response status codes:

OK- New user has been added

Bad request- No such user found

Internal server error- Something went wrong

/users/:id

Description: This endpoint allows users remove their account.Endpoint returns a confirmation message that users account has been deleted.

Query Parameters: :id,

Expected request data: Required: :id

Example:

localhost:8080/users/2


Expected response data: A confirmation message that users account has been deleted.

Example:

John Doe's account has been deleted!

Response status codes:

OK- Users account has been deleted

Bad request- No such user found!

Internal server error- Something went wrong

/users/:id/:movieTitle

Description:This endpoint allows users to add movie to their favorite list.Endpoint returns a confirmation message that movie has been successfully added.

Query Parameters: :id, :movieTitle

Expected request data: Required: :id, :movietitle

Example:

localhost:8080/users/1/New%20Movie

Expected response data: A confirmation message that movie has been added to the users list

Example:

Response status codes:

OK- Movie has been added to users list!

Bad request- No such user found!

Internal server error- Something went wrong
