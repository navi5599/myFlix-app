myFlix API Documentation

This is an REST API for application called myFlix that interacts with database and store data about different movies

You will be able to Sing up,update your personal information,and create a list of your favorite movies


/users

Description:This endpoint allows users to register.Endpoint returns a JSON object with few key values.For key "name" it returns string,for "favorite movies" array, and for an "id" string.

Query Parameters: None

Expected request data: JSON object with data about user. Required: "name,username,password,email"

Example:

{
"Name": "John Smith",
"Username": "jsmith",
"Password": "12345",
"Email": "jsmith@email.com",
}

Expected response data: JSON object with data about user which includes automatic generated ID

Example:

{
"Name": "John Smith",
"Username": "jsmith",
"Password": "12345",
"Email": "jsmith@email.com",
"favorite movies": [],
"id": "8a757bb9-2eec-46bf-86f0-443ff742cf3a"
}

Response status codes:

Created- New user has been added

Bad request- User already exists

Internal server error- Something went wrong

/users/:Username

Description: This endpoint allows users to update their info,such as Username.Endpoint returns a JSON object with few key values.For key "name" it returns string,for "favorite movies" array, and for an "id" integer.

Query Parameters: :Username,

Expected request data: JSON object with data about user. Required: :Username

Example:

localhost:8080/users/jsmith


{
"Username": "newUsername"
}

Expected response data: JSON object with data about user which includes,"id" with integer value, "Name","Username" and "Password" with string values and "favorite movies" with an array.

Example:

{
"id": 8a757bb9-2eec-46bf-86f0-443ff742cf3a,
"Name": "John Smith",
"Username": "newUsername",
"Password": "12345",
"Email": "jsmith@email.com",
"favoriteMovies": [
"Venom"
]
}

Response status codes:

OK- New user has been added

Internal server error- Something went wrong

/users/:Username

Description: This endpoint allows users remove their account by their Username.Endpoint returns a confirmation message that users account has been deleted.

Query Parameters: :Username,

Expected request data: Required: :Username

Example:

localhost:8080/users/jsmith


Expected response data: A confirmation message that users account has been deleted.

Example:

John Smith's account has been deleted!

Response status codes:

OK- Users account has been deleted

Bad request- Users name was not found

Internal server error- Something went wrong

/users/:Username/movies/:MovieID

Description:This endpoint allows users to add movie to their favorite list.Endpoint returns an JSON object with users data including updated favorite movies array.

Query Parameters: :Username, :MovieID

Expected request data: Required: :Username, :MovieID

Example:

localhost:8080/users/jsmith/movies/gh3v5jkb6h54h64vvg65

Expected response data: JSON object with updated list of movies in Favorite Movies array.

{
"id": 8a757bb9-2eec-46bf-86f0-443ff742cf3a,
"Name": "John Smith",
"Username": "newUsername",
"Password": "12345",
"Email": "jsmith@email.com",
"favoriteMovies": [
"Venom",
"New Movie" ]
}

Response status codes:

OK- Updated users array list

Internal server error- Something went wrong
